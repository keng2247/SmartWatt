🤖 Artificial Intelligence & Machine Learning Terms

Term What It Is — What It Does in SmartWatt
Artificial Intelligence (AI) Technology that makes computers learn and make decisions like humans. In SmartWatt, AI learns household electricity usage patterns from data and predicts how much each appliance consumes.
Machine Learning (ML) A branch of AI where a program learns from data without being explicitly programmed. SmartWatt uses ML to train 22 neural network models that predict appliance consumption.
Neural Network An AI model inspired by the human brain — made of connected layers of "neurons". SmartWatt uses one neural network per appliance (22 total) to predict efficiency factor and effective hours.
Deep Learning A type of ML that uses neural networks with multiple hidden layers. SmartWatt's multi-layer network (128→64 neurons) is a deep learning model.
Supervised Learning Training a model using labelled input-output pairs. SmartWatt trains with synthetic data where inputs = appliance specs and outputs = efficiency factor + usage hours.
Multi-output Neural Network A neural network that predicts two values at the same time from one input. SmartWatt uses this to predict both efficiency factor (0.5–1.5) and effective hours (0–24) in a single forward pass.
Hybrid AI–Physics Model SmartWatt's most important concept — combines AI (learns patterns) with physics formulas (sets real-world limits). This prevents unrealistic predictions while still capturing complex usage patterns.
Prediction / Inference Using a trained AI model on new, unseen data to get outputs. Every time a user submits appliances in SmartWatt, the system runs inference on each appliance's neural network.
Overfitting When a model memorises training data instead of learning general patterns — performs well on training data but badly on new data. SmartWatt prevents this using Dropout and Early Stopping.
Generalisation A model's ability to perform well on new, unseen data. SmartWatt generalises by training on diverse 12,000-household data with varied seasons, locations, and appliance specs.
Monte Carlo Simulation A technique that randomly samples from defined probability distributions many times to generate large, realistic datasets. SmartWatt uses it to generate 12,000 Kerala household electricity profiles for training.
Synthetic Dataset A dataset generated by computer simulation rather than collected from real measurements. SmartWatt uses a synthetic dataset because real KSEB meter data is not publicly available.
Feature A single measurable input variable given to a machine learning model. SmartWatt features include: ac_tonnage, star_rating, n_occupants, season, location_type.
Label / Target The output value a model is trained to predict. In SmartWatt, the labels are real_efficiency_factor and real_effective_hours for each appliance.
Confidence Score A percentage showing how much the system trusts its own prediction. SmartWatt starts at 98.2% and reduces it if anomalies are detected, fallback mode is used, or input features are missing.
Anomaly Detection Identifying data points that deviate significantly from normal. SmartWatt has two types: Usage Anomaly (hours exceed limits) and Efficiency Anomaly (appliance consuming 15%+ more than rated).
Bias (in AI) A systematic error in predictions. SmartWatt's BiasAdjuster service detects when predictions are consistently off for a user and applies a correction multiplier.
Self-Learning System A system that automatically improves using real production data over time. SmartWatt retrains its models every 6 hours using actual user data from Supabase.
Model Deployment Making a trained AI model available for live use. SmartWatt deploys updated models only if they pass MAE and R² validation — preventing regressions.

🧠 Neural Network Architecture Terms

Term What It Is — What It Does in SmartWatt
Neuron / Node The basic unit of a neural network — takes inputs, multiplies by weights, adds bias, and passes through activation function. SmartWatt's hidden layers have 128 and 64 neurons.
Layer A group of neurons at the same level in the network. SmartWatt has: Input Layer → Hidden Layer 1 (128) → Hidden Layer 2 (64) → Two Output Heads.
Input Layer The first layer that receives raw data. SmartWatt's input is appliance specs + household features (tonnage, star rating, season, occupants, etc.).
Hidden Layer Intermediate layers between input and output — learn complex patterns. SmartWatt has two: 128 neurons then 64 neurons, both with ReLU activation.
Output Layer / Head The final layer giving the prediction. SmartWatt has two output heads — one for efficiency factor, one for effective hours (multi-output design).
Shared Layers Hidden layers whose learned representations feed into multiple output branches. SmartWatt's 128→64 layers are shared between the efficiency and hours output heads.
Activation Function (ReLU) A function applied to each neuron's output — ReLU outputs 0 for negative values and the value itself for positives. Used in SmartWatt's hidden layers to add non-linearity and learn complex patterns.
Weight / Parameter Numbers inside a neural network that are adjusted during training to minimise error. SmartWatt's models update hundreds of weights per training step.
Dropout A regularization technique that randomly turns off 20% of neurons during each training step — prevents overfitting. SmartWatt applies Dropout(0.2) between hidden layers.
Forward Pass The process of feeding input data through the network to get an output/prediction. Every SmartWatt prediction is a forward pass through the appliance's neural network.
Backpropagation The process of sending error backwards through the network to update weights during training. Used in SmartWatt's training loop.
Loss Function (MSE) A formula that measures how wrong the model's predictions are. SmartWatt uses Mean Squared Error (MSE) — penalises larger errors more heavily.
Loss Weight A multiplier applied to individual output losses in a multi-output model. SmartWatt weighs efficiency loss 10× higher than hours loss because efficiency is harder to predict accurately.
Optimizer (Adam) An algorithm that adjusts weights to reduce loss. Adam (Adaptive Moment Estimation) is used in SmartWatt — it automatically adjusts the learning rate per parameter.
Learning Rate Controls how big each weight update step is during training. Too high = unstable, too low = slow. Adam handles this automatically in SmartWatt.

📊 Training & Evaluation Terms

Term What It Is — What It Does in SmartWatt
Epoch One complete pass through the entire training dataset. SmartWatt trains for up to 120 epochs, but Early Stopping usually stops it earlier when validation loss plateaus.
Batch Size Number of training samples processed in one step before weights are updated. SmartWatt uses batch size 32 — a balance between speed and stable learning.
Training Set Data used to train/learn model weights. SmartWatt uses 80% of the 12,000-household dataset for training.
Validation Set Data not seen during training, used to monitor performance after each epoch. SmartWatt uses the remaining 20% as the validation set.
Early Stopping A technique that stops training when validation loss doesn't improve for a set number of epochs. SmartWatt uses patience=10 — stops if no improvement for 10 consecutive epochs, then restores the best weights.
MAE (Mean Absolute Error) Average of absolute differences between predicted and actual values. SmartWatt's LearningPipeline uses MAE to judge if a newly trained model is better than the current one.
R² Score Measures how well the model explains variance in the data — 1.0 is perfect, 0 means no better than mean. SmartWatt requires a higher R² for the new model to replace the old one in production.
StandardScaler A scikit-learn tool that normalises numerical features to have mean=0 and standard deviation=1. SmartWatt scales numeric features like tonnage, capacity, occupants before feeding to the neural network.
OneHotEncoding Converts categorical variables into binary columns. SmartWatt encodes features like season (summer/monsoon/winter) and location_type (urban/rural) this way.
ColumnTransformer A scikit-learn pipeline tool that applies different preprocessing steps to different columns. SmartWatt uses it to apply StandardScaler to numerics and OneHotEncoder to categoricals simultaneously.
Preprocessor (.pkl) The saved scikit-learn ColumnTransformer fitted during training. Saved as a .pkl file alongside each model and used at inference time to transform new inputs identically.
Train-Test Split Dividing dataset into separate training and evaluation portions. SmartWatt uses 80/20 split — 9,600 for training, 2,400 for validation.
model.save() / .keras TensorFlow function to save a trained model to disk. SmartWatt saves each of the 22 models as appliance_model.keras files in the models/ folder.
joblib (.pkl) A Python library for efficiently saving and loading scikit-learn objects (like the preprocessor pipelines). Each appliance preprocessor is saved as appliance_preprocessor.pkl.
ThreadPoolExecutor A Python library for running tasks in parallel threads. SmartWatt uses it in BatchPredictor to predict all 22 appliances simultaneously — 3–5× faster than sequential.

🎯 SmartWatt Project-Specific Terms

Term What It Is — What It Does in SmartWatt
Efficiency Factor AI output 1 — ranges 0.5 to 1.5. Represents appliance degradation: 1.0 = rated performance, 1.2 = 20% more power (degraded), 0.8 = efficient. Used in the final kWh formula.
Effective Hours AI output 2 — ranges 0 to 24 hours/day. The model's learned estimate of how many hours this appliance actually operates daily given the household context.
Base Watts The rated electrical power of an appliance calculated by the Physics Engine from specs. e.g., 1.5T 3-star AC → 2160W. Multiplied with efficiency factor and hours to get kWh.
monthly_kWh formula The core calculation: (Base Watts × Efficiency Factor × Effective Hours × 30) / 1000. Dividing by 1000 converts Wh to kWh, multiplying by 30 gives monthly total.
System Load Unaccounted electricity from standby devices, phone chargers, routers, wiring losses — things the user didn't list. SmartWatt caps this at 15% of total consumption.
System Load Cap (15%) The maximum allowed unaccounted consumption. Based on electrical engineering studies. If System Load exceeds 15%, the excess is redistributed proportionally to major appliances.
Usage Anomaly First type of anomaly — triggered when usage hours exceed realistic limits (e.g., geyser > 3h/day, AC > 16h/day). Indicates unusual behaviour that the user should verify.
Efficiency Anomaly Second type — triggered when efficiency factor > 1.15 (Warning) or > 1.25 (Critical). Indicates the appliance is consuming significantly more than rated → may need servicing or replacement.
Hybrid Prediction SmartWatt's core approach — AI predicts the efficiency factor and hours, while Physics provides the base wattage. The final kWh is computed from both together.
Physics-Only Fallback If AI model fails to load or transform fails, SmartWatt falls back to pure physics calculation using rated specs. Confidence score is reduced when this happens.
Smart Feature Inference When a required input feature is missing, SmartWatt's predictor estimates it from other available data (e.g., estimates n_occupants from number of fans or AC units the user listed).
BiasAdjuster A service that applies a learned per-user correction multiplier to predictions. If a user's past predictions were consistently 10% too high, it applies a 0.9× factor next time.
What-If Simulation A feature that shows predicted kWh savings for 8 upgrade scenarios — upgrade AC star rating, switch to BLDC fans, replace old fridge, reduce geyser time, switch to LED lights, etc.
Appliance Preloading On server startup, all 22 models and 22 preprocessors are loaded into RAM so prediction requests are answered instantly without file loading delay.
distributeEnergyGap() Frontend function in energyUtils.ts. Intelligently distributes the difference between physics-calculated total and the user's actual bill kWh — scales appliances up or down to match the real bill.

🐍 Python & Backend Terms

Term What It Is — What It Does in SmartWatt
Python Primary programming language for SmartWatt's backend — chosen for its rich AI/ML library ecosystem (TensorFlow, scikit-learn, Pandas, NumPy).
TensorFlow Google's open-source ML framework for building and running neural networks. SmartWatt uses it to define model architecture, train models, save them, and run inference.
Keras High-level API built on top of TensorFlow that makes building neural networks simpler. SmartWatt uses keras.Model, keras.layers.Dense, and keras.layers.Dropout.
scikit-learn Python ML library for data preprocessing and evaluation. SmartWatt uses it for StandardScaler, OneHotEncoder, and ColumnTransformer.
Pandas (pd) Python library for data manipulation using DataFrames (tables). SmartWatt uses it to load and process the training CSV dataset and prepare input data for AI models.
NumPy (np) Python library for fast numerical computation using arrays. SmartWatt uses it for matrix operations, clipping values (e.g., capping efficiency between 0.5–1.5), and averaging training curves.
Pydantic Python library for data validation using type annotations. SmartWatt uses Pydantic models in schemas.py to validate all API inputs — invalid data is rejected before AI runs.
@validator Pydantic decorator for custom validation logic. SmartWatt uses it to enforce appliance-specific rules like "AC tonnage must be between 0.5 and 5.0 tons".
uvicorn A fast Python ASGI web server. SmartWatt runs uvicorn main:app --reload to start the backend — it handles incoming HTTP requests and routes them to FastAPI.
@app.on_event("startup") FastAPI decorator that runs a function when the server starts. SmartWatt uses it to call get_predictor().preload_all_models() — loads all 22 models into memory before any request arrives.
Singleton Pattern A design pattern ensuring only one instance of a class exists. SmartWatt's predictor uses get_predictor() which returns the same AppliancePredictor instance across all requests.
CORS Middleware Cross-Origin Resource Sharing — allows the frontend (on a different domain/port) to make API calls to the backend. SmartWatt configures CORS with allow_origins=["*"] for open access.
Joblib Python library for saving/loading Python objects efficiently. SmartWatt uses joblib.dump() to save preprocessors as .pkl files and joblib.load() to restore them.
python-dotenv Library that loads environment variables from a .env file into Python's environment. SmartWatt uses it to load Supabase URL and API key without hardcoding secrets.
EarlyStopping callback A Keras callback that monitors validation loss during training and stops when no improvement is seen for patience epochs. SmartWatt uses patience=10, restore_best_weights=True.

🌐 FastAPI & API Terms

Term What It Is — What It Does in SmartWatt
FastAPI Modern Python web framework for building REST APIs — fast, automatic documentation, built-in validation. SmartWatt's entire backend is built on FastAPI.
REST API Representational State Transfer — a standard for web APIs using HTTP methods (GET, POST). SmartWatt's backend exposes REST endpoints that the Next.js frontend calls.
Endpoint A specific URL in the API that performs an action. SmartWatt endpoints: /predict-appliance, /predict-all, /calculate-bill, /simulate-savings, /health.
HTTP POST An HTTP method that sends data to the server in the request body. SmartWatt's prediction and bill calculation endpoints all use POST requests with JSON body.
HTTP GET An HTTP method that retrieves data from the server. SmartWatt's /health and / (root) endpoints use GET.
JSON (JavaScript Object Notation) Lightweight data format used for sending data between frontend and backend. All SmartWatt API requests and responses use JSON format.
Request Body Data sent along with a POST request. SmartWatt sends appliance details, household info, and usage data as JSON in the request body to /predict-all.
Response What the server sends back after processing a request. SmartWatt responses include: prediction (kWh), insights (efficiency, hours), confidence, and anomalies.
Router A FastAPI component that groups related endpoints together. SmartWatt has a routers/appliances.py router that handles all prediction and simulation endpoints.
Axios A JavaScript HTTP client library used in the frontend to make API calls. SmartWatt's api/client.ts configures Axios with the backend URL and timeout.
/health endpoint An API endpoint that confirms the server is running and models are loaded. SmartWatt returns {"status":"healthy", "models_loaded":22} — useful for deployment monitoring.
Status Code 200 HTTP success code — means the request was processed correctly. SmartWatt returns 200 for successful predictions.
Status Code 422 HTTP Unprocessable Entity — returned by FastAPI/Pydantic when input validation fails. SmartWatt returns 422 with error details if AC tonnage is out of range, for example.

⚡ Electrical & Physics Terms

Term What It Is — What It Does in SmartWatt
Watt (W) The unit of electrical power — how fast energy is consumed. SmartWatt's Physics Engine converts appliance specs into watts (e.g., 1.5T AC = 2160W).
Kilowatt (kW) 1 kilowatt = 1000 watts. Used to express larger power values. A 2kW geyser = 2000W.
kWh (Kilowatt-hour) The unit of energy on your KSEB electricity bill — 1 kWh = using 1000W for 1 hour. SmartWatt predicts monthly kWh consumption for each appliance individually.
Power Rating The maximum electrical power an appliance is designed to consume, printed on the label (e.g., "750W"). SmartWatt's Physics Engine uses power ratings as the baseline for calculations.
Star Rating Energy efficiency rating given by BEE (Bureau of Energy Efficiency) — 1 star (least efficient) to 5 stars (most efficient). Higher star = lower electricity consumption. SmartWatt uses star rating to adjust base watts.
BEE (Bureau of Energy Efficiency) Government body that assigns star ratings to appliances in India. SmartWatt's physics formulas are based on BEE efficiency standards.
Tonnage (AC) Measure of an AC's cooling capacity — 1 ton cools approximately 100 sq ft. SmartWatt supports 1.0T, 1.5T, and 2.0T ACs — higher tonnage = higher watts.
HP (Horsepower) Unit of power for motors like water pumps — 1 HP = 746 Watts. SmartWatt uses HP × 746 to calculate water pump wattage.
BLDC (Brushless DC) Fan An energy-efficient fan type using brushless DC motor — consumes only 30W vs 75W for standard fans. SmartWatt's simulation shows users the savings of switching to BLDC.
Voltage Fluctuation Variation in electrical supply voltage — common in rural Kerala. Causes appliances to consume more power or degrade faster. SmartWatt's synthetic dataset includes rural voltage fluctuation as a factor.
Standby Power Electricity consumed by devices when switched off but still plugged in (e.g., TV on standby). Part of SmartWatt's System Load which is capped at 15%.
Efficiency Degradation Over time, appliances become less efficient and consume more electricity than their rated value. SmartWatt models this via the efficiency factor — older appliances get higher factors.
Inverter AC An AC with variable-speed compressor — more efficient at partial loads. In SmartWatt, 5-star ACs are treated as inverter-type with lower effective wattage.
Wiring Loss / Transmission Loss Electricity lost as heat in the home's wiring — typically 2–5% of total consumption. Part of SmartWatt's system load accounting.
FSM (Fuel Surcharge Mechanism) An additional charge on every unit of electricity to cover fuel cost variations. KSEB charges ₹0.13 per unit as FSM — included in SmartWatt's bill calculation.

🏛️ KSEB & Billing Terms

Term What It Is — What It Does in SmartWatt
KSEB Kerala State Electricity Board — the organisation that generates, distributes, and bills electricity in Kerala. SmartWatt is specifically designed around KSEB's tariff structure.
Tariff The pricing structure for electricity — how much each unit costs. KSEB uses different tariffs for domestic, commercial, and industrial consumers. SmartWatt uses the Domestic LT-1A tariff.
Telescopic Tariff A slab-based pricing where different units are charged at different rates that increase with consumption. SmartWatt implements all 5 slabs for ≤250 units: ₹3.25, ₹4.05, ₹5.10, ₹6.95, ₹8.20 per unit.
Slab (Tariff Slab) A range of units charged at the same rate. KSEB slabs are each 50 units wide (0–50, 51–100, etc.). In SmartWatt for ≤250 units: first 50 units @ ₹3.25, next 50 @ ₹4.05, and so on.
Flat Rate Tariff When consumption exceeds 250 units/month, KSEB charges a single flat rate on the entire consumption. SmartWatt's tariff_config.py stores these flat rates (₹6.40 for ≤300 units, higher above that).
Bi-Monthly Billing KSEB bills every 2 months (every 60 days). SmartWatt multiplies monthly calculations by 2 when showing the estimated bill to match actual KSEB invoices.
Energy Charge The main component of the electricity bill based on units consumed × tariff rate. SmartWatt calculates this using KSEB slabs and doubles it for bi-monthly format.
Fixed Charge A flat fee independent of usage — KSEB charges this based on connected load category. SmartWatt's current implementation focuses on the energy charge portion.
Units (kWh) What the electricity meter measures — each "unit" shown on your KSEB bill = 1 kWh. SmartWatt converts all appliance predictions to monthly units and then calculates the bill.
LT-1A (Domestic) KSEB tariff category for domestic residential consumers (Low Tension supply, Category 1A). SmartWatt's tariff calculation is specifically for this category.
KSEB Tariff 2024-25 The official tariff rates approved by KSEB for the year 2024–2025. SmartWatt uses these exact rates: ₹3.25/unit (0–50), ₹4.05 (51–100), ₹5.10 (101–150), ₹6.95 (151–200), ₹8.20 (201–250).

🗄️ Database & Storage Terms

Term What It Is — What It Does in SmartWatt
Supabase Open-source Firebase alternative — provides PostgreSQL database, authentication, and storage as a service. SmartWatt uses it for user data, appliance details, and prediction storage.
PostgreSQL A powerful, open-source relational database system. Supabase runs on PostgreSQL — it stores SmartWatt's user records, predictions, and training data in structured tables.
JSONB PostgreSQL's binary JSON storage format — stores JSON data efficiently and supports fast querying. SmartWatt uses JSONB to store appliance details (each user picks different appliances with different fields).
Schema The structure/definition of a database — what tables exist and what columns they have. SmartWatt's schema has a smartwatt_training table with columns for user_id, household data, appliance JSONB, and predictions JSONB.
Upsert Database operation that inserts a new record if it doesn't exist, or updates it if it does. SmartWatt uses upsert so each user has one record that is updated each time they analyse.
Unique Constraint A database rule ensuring a column has no duplicate values. SmartWatt's user_id column has a unique constraint — one prediction record per user.
Row Level Security (RLS) A Supabase/PostgreSQL feature that restricts which rows a user can read/write. SmartWatt uses RLS to ensure users can only access their own data.
Anon Key Supabase's public API key — safe to include in frontend code. Used by SmartWatt's frontend to connect to Supabase. Unlike the service key, it respects RLS policies.
Service Role Key Supabase's admin key with full database access — bypasses RLS. SmartWatt's backend uses this to write prediction data and trigger learning pipeline.
Real-time Subscription Supabase feature that pushes database changes to connected clients instantly. SmartWatt could use this for live updates (a future scope item).
Authentication (Auth) Verifying user identity. SmartWatt uses Supabase Auth (email/password) — users must log in before accessing their electricity analysis.
Session A temporary authenticated state after login. SmartWatt's frontend checks for an active Supabase session on startup and restores saved user data if found.

💻 General Computer Science Terms

Term What It Is — What It Does in SmartWatt
Full-Stack Application An application with both frontend (user interface) and backend (server + database). SmartWatt is a full-stack app: Next.js frontend + FastAPI backend + Supabase database.
Service-Oriented Architecture Designing software as a collection of separate, reusable services each handling one responsibility. SmartWatt's backend uses this — InputNormalizer, BatchPredictor, BiasAdjuster, etc. are separate services.
Modular Design Organising code into independent, reusable modules. SmartWatt separates concerns into routers, services, engines, utilities — each file has one clear job.
Parallel Processing Running multiple tasks simultaneously instead of one after another. SmartWatt uses Python's ThreadPoolExecutor to predict all 22 appliances at the same time — 3–5× faster.
Graceful Degradation System continues working (in a reduced capacity) when a component fails. SmartWatt falls back to physics-only calculation if the AI model fails to load.
Input Validation Checking that user inputs are correct before processing. SmartWatt validates all API inputs via Pydantic schemas before they reach the AI model.
Error Handling (try/except) Code that catches and manages runtime errors gracefully. SmartWatt wraps each simulation scenario in independent try/except blocks so one failure doesn't block others.
Environment Variable A value stored outside the code, loaded at runtime — keeps secrets like API keys safe. SmartWatt stores Supabase URL and keys in .env / .env.local files.
Docker A platform for packaging applications into portable containers so they run identically everywhere. SmartWatt has a Dockerfile for containerised backend deployment.
Docker Compose Tool for defining and running multi-container Docker applications. SmartWatt's docker-compose.yml starts both frontend and backend together.
Singleton Design pattern ensuring a class has only one instance. SmartWatt's AppliancePredictor is a singleton — one shared instance holds all 22 loaded models to save memory.
Data Persistence Saving data so it survives beyond the current session. SmartWatt persists user inputs and predictions to Supabase so users can return and see their previous analysis.
Responsive Design UI that adapts to different screen sizes (desktop, tablet, mobile). SmartWatt's frontend uses Tailwind CSS responsive utilities for good mobile and desktop experience.
CDN (Content Delivery Network) A network of servers that deliver web content from the location closest to the user — faster loading. Vercel deploys SmartWatt's frontend on a global CDN.
Transformer (ML preprocessing) Also refers to the scikit-learn ColumnTransformer that applies different preprocessing to different columns. SmartWatt's preprocessor saved as .pkl is a fitted ColumnTransformer.
API Documentation (Swagger) FastAPI automatically generates interactive API documentation at /docs — shows all endpoints, their inputs, and outputs. SmartWatt's backend docs are available at http://localhost:8000/docs.
